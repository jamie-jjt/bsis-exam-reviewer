BSIS Entrance Review Guide

A comprehensive guide covering logic, mathematics, and information technology fundamentals.

Logical Reasoning & Mathematics

Entrance exams for Information Systems evaluate your ability to recognize patterns and solve quantitative problems. These skills are fundamental to algorithm design and data analysis.

Pattern Recognition

Sequence: 2, 4, 8, 16, ? (Answer: 32). The pattern is geometric progression (multiplying by 2).

Syllogism: All servers are computers. All computers require power. Therefore, all servers require power.

Algebra & Percentages

Equation: 3x + 6 = 18. Subtract 6 (3x=12), then divide by 3 (x=4).

Discount: A ₱1,000 item with a 20% discount costs ₱800 (1000 - (1000 * 0.2)).

Graphs & Data Representation

Graphs are visual ways to show data so it's easier to understand. In exams, you'll often be asked to interpret what a graph is saying or choose the best type for a specific scenario.
  • Bar Graph: Best for comparing different categories (e.g., Number of students in IT vs. CS vs. IS).
  • Line Graph: Best for showing trends over time (e.g., How computer prices have changed from 2010 to 2025).
  • Pie Chart: Best for showing parts of a whole or percentages (e.g., Percentage of mobile users vs. PC users).

Exam Tip: Interpretation

Think of it like a story: If a line graph is going up, something is increasing. If a slice of a pie chart is more than half, it represents more than 50% of the total. Always check the labels on the X (horizontal) and Y (vertical) axes first!

Computer Hardware & Architecture

Hardware refers to the tangible parts of a computer system. Understanding the internal architecture helps in identifying how data moves through a system.
  • CPU (Central Processing Unit): The "Brain" that executes instructions.
  • RAM (Volatile Memory): Temporary storage for active programs; data is lost when power is cut.
  • Storage (HDD/SSD): Non-volatile memory where files are stored permanently.
  • Input/Output (I/O): Input devices (Keyboard, Mouse) bring data in; Output devices (Monitor, Printer) send data out.

Key Concept: The Fetch-Execute Cycle

The CPU continuously fetches an instruction from memory, decodes what it means, and executes it. This cycle happens billions of times per second (measured in GHz).

Software & Operating Systems

Software is the set of instructions that tells the hardware what to do. The Operating System (OS) acts as the intermediary between the user and the hardware.
  • System Software: Includes the OS (Windows, Linux, macOS) and utility programs.
  • Application Software: Programs for users (Word processors, Browsers, Games).
  • Firmware: Permanent software programmed into a hardware device (like the BIOS).

OS Functions

Resource Management: Allocating CPU time and memory to different apps.

User Interface: Providing a way for humans to interact (GUI or CLI).

Networking & Connectivity

Networking is the practice of connecting computers together to share resources. To understand it for the exam, you must know how they are connected (Topologies) and the rules they follow (OSI Model).

Network Topologies

A topology is the "map" or layout of how devices are connected.

Bus Topology

Definition: All devices share a single main cable (the "bus").

Advantage: Very cheap and easy to set up for small rooms.

Disadvantage: If the main cable breaks, the whole network dies.

Example: A small, old office setup.

Star Topology

Definition: All devices connect to a central hub or switch.

Advantage: If one cable breaks, only that computer goes offline.

Disadvantage: If the central hub fails, everyone loses connection.

Example: Home WiFi routers or modern Internet Cafés.

Ring Topology

Definition: Devices connect in a circle; data travels in one direction.

Advantage: No "data collisions" because everyone takes turns.

Disadvantage: Like a string of Christmas lights—if one node fails, the whole ring breaks.

Example: Some specialized fiber-optic campus networks.

Mesh Topology

Definition: Every device is connected to every other device.

Advantage: Extremely reliable (many backup paths).

Disadvantage: Very expensive and complex to wire up.

Example: The core of the Global Internet.

Tree & Hybrid

Tree: A hierarchy (like a family tree) where branches connect to a "root."

Hybrid: A mix of two or more topologies (e.g., several Stars connected by a Bus).

Example: Large university or corporate building networks.

The OSI Model (7 Layers)

The OSI Model is a standard that explains how data moves from one computer to another. Use this mnemonic to remember the order from Bottom to Top:

"Please Do Not Throw Sausage Pizza Away"

# Layer What it does (Simple English) Example / Analogy
7 Application The window where the user interacts. Web Browser (Chrome), Email (Gmail)
6 Presentation Translates data so the app can read it. Encryption, JPG, MP4 formats
5 Session Starts and stops the "conversation." The "Hello" and "Goodbye" of a call
4 Transport Breaks data into pieces and ensures they arrive. TCP (Checking if packets arrived)
3 Network Finds the best path (Routing). IP Address / Routers
2 Data Link Connects two physically near devices. MAC Address / Ethernet Switches
1 Physical The actual wires, radio waves, or light. Cables, Hubs, WiFi signals

Programming Logic & Algorithms

Programming isn't just about typing code; it's about Step-by-Step Thinking (Algorithms). Think of an algorithm like a recipe for baking a cake.

1. Variables: The Containers

Think of a variable like a labeled box. You put data inside to use later.

  • Numeric: Stores numbers you can do math with (e.g., age = 18).
  • Text (String): Stores words or characters (e.g., name = "Juan").

2. Conditions: The Fork in the Road

IF–ELSE logic: Computers make decisions based on whether something is TRUE or FALSE.

Example: IF you have ₱100, THEN buy a burger; ELSE (otherwise) buy a biscuit.

3. Loops: The Repeat Button

Why write the same instruction 100 times? A Loop tells the computer to repeat a task until it's finished.

Example: "While there are still students in the line, check their IDs."

Database Management & SQL

A database is like a digital filing cabinet. While Excel is good for small lists, Databases are better for huge amounts of data because they are faster, safer, and can link different types of information together.

The Structure: Like a Class Record

  • Table: The whole spreadsheet (e.g., "Student Roster").
  • Row (Record): One specific student (e.g., "John Doe's entire info").
  • Column (Field): One specific detail (e.g., "Phone Number" or "Birthday").

Primary vs. Foreign Keys

Primary Key: A unique ID that identifies a record. (Example: Your Student ID number—no two students have the same one).

Foreign Key: An ID in one table that points to a Primary Key in another table to link them together.

What SQL Looks Like

SQL is just "asking questions" to the database. In an exam, you might see something like:

SELECT name FROM Students WHERE grade = 'A';

What does this mean? "Show me the names of all the students who have an A grade."

Information Systems Concepts

Information Systems (IS) go beyond technology. They involve People, Process, and Technology working together to manage information for organizational goals.
  • TPS (Transaction Processing System): Records daily routine transactions (e.g., a Point-of-Sale system).
  • MIS (Management Information System): Provides reports for middle management to monitor performance.
  • DSS (Decision Support System): Helps high-level managers solve non-routine, complex problems.
  • ERP (Enterprise Resource Planning): A massive software suite that integrates all departments (Finance, HR, Sales) into one database.

Cybersecurity Fundamentals

As an IS student, you must understand how to protect data. Security is often described by the CIA Triad:
  • Confidentiality: Only authorized people can see the data (Encryption).
  • Integrity: Data is accurate and hasn't been tampered with.
  • Availability: Systems are up and running when needed.

Common Threats

Phishing: Fake emails designed to steal passwords.

Malware: Malicious software like viruses, worms, or ransomware.

Web Technologies & Cloud

Modern Information Systems are almost always web-based. This involves the interaction between a client (user's browser) and a server.
  • HTML: The structure of a webpage (The skeleton).
  • CSS: The styling and layout (The clothes/makeup).
  • Cloud Computing: Storing and accessing data/programs over the internet instead of your computer's hard drive (e.g., AWS, Google Drive).
